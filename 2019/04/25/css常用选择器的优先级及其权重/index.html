css常用选择器的优先级及其权重

发表于 2019-04-25 | 更新于 2019-04-28 | 分类于 web前端

简语

在前期学习css的过程中，我们会接触到许多的选择器，比如我们最为常用的class选择器、标签选择器等等……
相信对于这些选择器的使用和优先级，小伙伴们也是十分的熟悉吧，然而关于它们到底是按一个什么方式去进行比较优先级的呢？
首先先让我们来复习一下几个最常用选择器以及他们的优先级以及来了解一下这些常用选择器的权重值吧.

通配符选择器

就是我们常见的*{}，通配符选择器的优先权非常的低
通配符选择器一般用于初始化：如对字体的颜色、大小，以及去掉body原有的margin，ul原有的小圆点等……

通配符选择器会对整个html文档的里面的所有便签都进行操作，十分的耗费资源，所以要谨慎使用，它的权重值则为0。

标签选择器(又称派生选择器)及伪元素

1.标签选择器
标签选择器也可以叫派生选择器，它的形式如div{}、p{}、span{}等……它会选中文档中多有符合条件的标签，所以，如果你想准确的选中某个标签的话，请在他的前面加上准确的父级选择器。
2.伪元素
伪元素是一个附加至选择器末的关键词，允许你对被选择元素的特定部分修改样式，伪元素选择器多用于去除浮动，或者在标签内容的开头和结尾插入一些元素，像一些小箭头等；它的形式如：div::after div::before 下面是我们利用伪元素::after去除浮动wrapp末尾的浮动流，使得wrapp后面的元素不会脱离文档流往上窜：

标签选择器和伪元素选择器的优先级是想等的，所以我们应该可以猜到他们的权重值应该也是一样的，它们的权重值则为1。

class选择器、属性选择器、伪类选择器

1.class选择器
class选择器是我们日常编程里面最为常用的选择器了，可以通过.xxx{}的方式来选中一个或多个标签，一个元素可以拥有多个classs名，多个元素可以拥有相同的class类名。
2.属性选择器
属性选择器可以获取我们自定义属性名的元素，同时也能通过a标签、img标签中的索引来获取该标签。
3.伪类选择器

id选择器

用来为标有特定id的html元素指定特定的样式，CSS中id选择器以”#”来定义。id选择器类似于我们的身份证号码。标签和id是一对一的关系。
id的权重值为100，在日常开发中，我们尽量少用id去标识标签，因为在后续和后端对接的过程中，后端有可能给我们的部分元素加上id，如果我们通过id去操作元素的话很有可能会出现bug

行间样式

行间样式顾名思义是在我们标签中加入style属性去写入css样式，这样的操作就像我们去纹身一样所以他的优先级也十分的高，它的权重值为1000。
后续我们通过js、jq选中某个元素进行操作也是操作它的行间样式，但在实际开发过程中，我们尽量不要在html的标签中去加入行间样式，避免在css渲染的过程中因为优先级的原因出现不必要的麻烦

!important

!important并不是一个选择器，它是一个加在css样式后面的属性，在css样式后面加上!important，那么该条样式就会变成最高优先级。!important它的权重值为infinity也就是无穷的意思，它的优先级是最大的。

总结

最后让我们来总结一下所有选择器的权重值：

选择器 权重值
!important(加在属性的后面) infinity
Id 100
class 属性选择器 伪类 10
标签选择器 伪元素 1
通配符 0

需要注意的一点是：这些权重值并不是十进制的，而是256进制的数
知道了这些权重值之后，就算在复杂的选择器组合我们都可以通过权重值相加的方式快速知道他们的优先级，以免产生冲突
